# 今日の課題: Textual製 Git Branch Manager

Python と Textual を用いて、CLI 作業中に使える Git ブランチ管理ツールを作成せよ。

## 概要
このツールは、現在のカレントディレクトリを Git リポジトリとして扱い、
ブランチの一覧表示・チェックアウト・削除・新規作成を行う TUI アプリである。

Git コマンドは、普段ターミナルでカレントディレクトリ上に対して実行するのと同様に、
このアプリを起動したディレクトリに対して作用するものとする。

## 必須要件

### 1. ブランチ一覧表示
- ローカルブランチ一覧を取得して表示すること
- 現在のブランチが一目でわかること
- 一覧は起動時に読み込まれること

### 2. チェックアウト機能
- 選択中のブランチへ切り替えられること
- 切り替え成功後、一覧表示を更新すること
- エラー時はメッセージを表示すること

### 3. 新規ブランチ作成機能
- 新しいブランチ名を入力できること
- 作成後は一覧を更新すること
- 同名ブランチが存在する場合はエラー表示すること

### 4. ブランチ削除機能
- 選択中ブランチを削除できること
- 削除前に確認ダイアログを表示すること
- 現在チェックアウト中のブランチは削除不可とすること
- 削除成功後は一覧を更新すること

### 5. Git リポジトリ判定
- カレントディレクトリが Git リポジトリではない場合、
  その旨を画面上に表示して通常操作を無効化すること

## 画面要件
- 左側: ブランチ一覧
- 右側: 選択中ブランチの詳細情報
- 下部または上部: 操作ヒント
- ボタン:
  - Checkout
  - Delete
  - New Branch
  - Refresh

## 詳細情報欄の表示例
- ブランチ名
- 現在のブランチかどうか
- 必要なら最新コミットの要約

## 例外処理
- Git コマンド失敗時にアプリが落ちないこと
- エラーメッセージをユーザーに表示すること
- 空文字や不正なブランチ名をそのまま実行しないこと

## 余力があれば
- リモートブランチ表示
- 検索機能
- 未マージブランチを色分け表示
- 強制削除機能
- 最新コミット日時順で並び替え

## 使用技術の想定
- Python
- textual
- subprocess
- pathlib

# 関連モジュールのポイント

## textual
TUI の画面・ウィジェット・イベント処理を担当する。
一覧表示には ListView や OptionList 系、
操作には Button、
入力には Input、
確認ダイアログには ModalScreen が使いやすい。

## subprocess
Git コマンドの実行に使う。
例:
- git branch
- git switch <name>
- git branch -d <name>
- git switch -c <name>

標準出力と標準エラー出力を受け取り、
成功/失敗を判定できるようにするのが重要。

## pathlib
カレントディレクトリの扱いやパス操作を安全に書くのに便利。

## 実装上のコツ
- Git操作部分は関数に分離する
  - list_branches()
  - checkout_branch(name)
  - create_branch(name)
  - delete_branch(name)
- UI と Git 実行処理を分けると見通しがよい
- まずは「一覧表示と更新」だけ完成させてから、
  checkout → create → delete の順で広げると作りやすい
