# 今日の課題（Java）: 「Undo付きミニテキストエディタ」

## 問題
次の操作を処理する、簡易テキストエディタを実装してください。  
初期状態の文字列 `S` は空です。

与えられる操作は以下の3種類です：

- `TYPE x` : 文字列 `S` の末尾に、文字列 `x` を追加する（xは英小文字のみ、長さ1以上）
- `BACK k` : 文字列 `S` の末尾から `k` 文字削除する（`k <= |S|` は保証）
- `UNDO` : 直前の操作（`TYPE` または `BACK`）を1回取り消して、取り消し前の状態に戻す  
  - `UNDO` 自体は履歴に積まない（＝連続UNDOは「1つ前の実操作」をさらに戻す）

最後に完成した `S` を出力してください。

## 入力
1行目: `Q`（操作回数）  
2行目以降: `Q` 行の操作

## 出力
最終的な `S` を1行で出力

## 制約（雑に安全側）
- `1 <= Q <= 2 * 10^5`
- `TYPE` の合計文字数も最大 `2 * 10^5` 程度
- 実行時間はそれなりに速く

## 例
入力:
```
6
TYPE abc
BACK 1
TYPE z
UNDO
UNDO
TYPE xy
```
出力:
```
abcxv
```

## 実装のヒント（最短ルート）
- `StringBuilder` を現在の文字列として使う
- 履歴スタックに「戻すための情報」を積む  
  - `TYPE x` のとき：追加した長さ `len(x)` を記録して、UNDO時に末尾 `len` を削る  
  - `BACK k` のとき：削除した文字列（末尾k文字）を記録して、UNDO時にそれを末尾へ再追加
- `UNDO` は履歴スタックから1つ取り出して復元するだけ

## クリア条件
- `Q` が大きくても TLE しない（O(Q + 総文字数) を意識）
- `UNDO` の仕様（UNDOを履歴に積まない）が守れている
