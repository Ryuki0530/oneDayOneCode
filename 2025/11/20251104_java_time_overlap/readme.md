# 🧩 今日の課題：区間重なり検出プログラム

## 概要
複数の時間帯（開始時刻と終了時刻）を入力として受け取り、  
それらの区間が「重なっているかどうか」を判定するプログラムを作成してください。

---

## 入出力仕様

### 入力
標準入力から以下の形式で与えられます：

N
start_1 end_1
start_2 end_2
...
start_N end_N



- `N`：区間の数（1 ≤ N ≤ 100）
- 各 `start_i`, `end_i`：整数（0 ≤ start_i < end_i ≤ 1000）  
  ※単位は分で構いません。

### 出力
重なっている区間が存在する場合は `"Overlap"`  
全く重なっていない場合は `"No Overlap"` と出力してください。

---

## 入出力例

### 入力例1
3
0 10
15 30
35 50



### 出力例1
No Overlap


---

### 入力例2
3
0 10
8 15
20 30


### 出力例2
Overlap
Overlap Duration: 2


---

## ヒント
- 区間 `[a, b]` と `[c, d]` が重なっている条件は次の式で判定できます：
a < d && c < b

- `List` に区間を格納し、二重ループで全組み合わせをチェックすると簡単です。
- 時間があれば、`Comparator` を使ってソート後に一度のループで判定する高速版にも挑戦してみましょう。

---

## 発展課題（時間がある人向け）
- 区間が重なる部分の「合計時間」を出力するように拡張してください。  
例：`[0,10]` と `[8,15]` の場合 → 重なり時間は `2` 分。

---

## 実行例（発展課題込み）

### 入力
3
0 10
8 15
20 30

### 出力
Overlap
Overlap Duration: 2
