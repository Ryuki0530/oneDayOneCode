# 今日のJava課題: Access Window Counter

## 問題
アクセスログが時刻順に与えられます。  
各ログは「時刻」と「ユーザーID」からなります。

各ユーザーについて、**直近 W 秒以内のアクセス回数**を管理しながら、
指定された時刻でのアクセス数を出力してください。

## 入力
以下の形式で標準入力から与えられます。

```text
N W
t1 user1
t2 user2
...
tN userN
```

- 1 行目:
  - `N` はログ件数
  - `W` は集計対象の秒数
- 続く `N` 行:
  - `ti` はアクセス時刻（整数）
  - `useri` はユーザーID（英小文字・数字のみ）

## 出力
各ログについて、そのアクセスを追加した後の  
**そのユーザーの直近 W 秒以内のアクセス数**を 1 行ずつ出力してください。

## 集計ルール
ある時刻 `t` のアクセスに対して、  
**時刻が `t - W + 1` 以上 `t` 以下**のアクセスを「直近 W 秒以内」とします。

## 制約
- `1 <= N <= 200000`
- `1 <= W <= 10^9`
- `0 <= ti <= 10^9`
- 入力は時刻昇順で与えられる
- 同じ時刻のアクセスが複数存在してもよい

## 入力例1
```text
7 5
1 alice
2 bob
3 alice
6 alice
7 bob
7 alice
10 alice
```

## 出力例1
```text
1
1
2
2
2
3
3
```

## 説明
時刻 6 の `alice` については、直近 5 秒は `[2, 6]` です。  
この範囲にある `alice` のアクセスは時刻 3 と 6 の 2 回なので、`2` を出力します。

時刻 10 の `alice` については、直近 5 秒は `[6, 10]` です。  
この範囲にある `alice` のアクセスは時刻 6, 7, 10 の 3 回なので、`3` を出力します。

## 要求
- Java で実装してください。
- 大きな入力でも高速に動作するようにしてください。

## ヒント
- ユーザーごとに「まだ有効なアクセス時刻」を管理すると考えやすいです。
- `Queue` や `Deque` が使えると便利です。
- ユーザーごとにキューを持つ方法を考えてみましょう。

## 発展
余裕があれば、最後に以下も出力するよう拡張してみてください。

- 全ユーザーの中で最大の直近アクセス数
- その最大値を持つユーザーID（複数なら辞書順最小）
