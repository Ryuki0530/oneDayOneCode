# 今日のC課題：固定長ログの「最新N件」表示（リングバッファ）

## 目的
標準入力で与えられるログ文字列（1行＝1ログ）を読み取り、**最後のN件だけ**を入力順に出力するツールを作ってください。  
（メモリを無駄にせず、固定長の配列で回す＝リングバッファ練習）

## 入力形式
- 1行目：整数 `N`（保持する件数）
- 2行目：整数 `M`（ログの総数）
- 3行目以降：ログ文字列が `M` 行（空白を含まない想定、最大長は `L` とする）

## 出力形式
- 最新の `min(N, M)` 件を、**古いものから順に**1行ずつ出力する。

## 制約（目安）
- `1 <= N <= 100000`
- `0 <= M <= 200000`
- 1ログの長さ `1 <= L <= 200`（改行除く）

## 例
入力:
```
3
7
A
B
C
D
E
F
G
```

出力:
```
E
F
G
```

## 実装要件
- `char buf[N][L+1]` 相当の固定長配列（または `malloc` で確保）を使う
- `head`（次に書く位置）を `head = (head + 1) % N` で回す
- 最後に「どこから何件出すか」を計算して出力する

## ヒント
- `count = min(M, N)`
- 最初に出力するインデックス：
  - `start = (head - count + N) % N`
- `count` 回だけ `idx = (start + i) % N` で辿る

## 追加チャレンジ（余裕があれば）
- ログに空白が入るケースにも対応（`fgets`で読む）
- `N` が大きいときにスタックに置かず `malloc` で確保する
