# Daily Java Challenge - Weather Carry Assistant

## 概要
無料APIを使って、**指定した地名の天気を取得し、持ち物アドバイスを返すCLIツール**を作成してください。

たとえば `Tokyo` と入力したら、位置情報を取得し、その地点の現在天気や降水確率をもとに、

- 傘が必要か
- 上着が必要か
- 暑さ対策が必要か

を表示するツールです。

---

## 作るもの
コマンドラインで地名を受け取り、次のような情報を表示するプログラムを作ってください。

### 入力例
```text
Tokyo
```

### 出力例
```text
場所: Tokyo
気温: 8.4℃
降水確率: 70%
風速: 12.1 km/h

持ち物アドバイス:
- 傘を持っていきましょう
- 少し暖かい服装がおすすめです
- 風が強いので注意してください
```

---

## 使用するAPI
### 1. 地名検索
地名から緯度・経度を取得する。

### 2. 天気予報
取得した緯度・経度を使って現在または今日の天気を取得する。

---

## 必須要件
1. ユーザーが地名を1つ入力できる
2. APIを呼び出してその地名の緯度・経度を取得する
3. 緯度・経度を使って天気情報を取得する
4. 気温・降水関連情報・風速を表示する
5. 条件に応じて持ち物アドバイスを出す

---

## アドバイス判定ルール例
以下は一例です。必要なら調整して構いません。

- 降水確率が50%以上 → `傘を持っていきましょう`
- 気温が10℃未満 → `暖かい服装がおすすめです`
- 気温が28℃以上 → `飲み物を持っていきましょう`
- 風速が10km/h以上 → `風が強いので注意してください`

---

## 余裕があれば
- 地名が見つからなかったときのエラー処理
- 複数候補がある場合に先頭候補を使う
- GUI化
- 今日だけでなく「明日の天気」も出す
- アドバイスを3段階評価にする
- 実行結果をテキストファイルに保存する

---

## 想定技術
- `java.net.http.HttpClient`
- `HttpRequest`
- `HttpResponse`
- JSONパース
- 条件分岐
- メソッド分割
- 例外処理

---

## 補足
APIレスポンスはJSONで返ってくるため、JavaでJSONを扱う方法を調べながら進めてください。

標準ライブラリだけで無理に頑張るより、JSONライブラリを1つ使ってもOKです。

---

## 目標
「APIを呼ぶだけ」で終わらず、**取得したデータを使ってユーザーにとって意味のある出力を返すこと**を目指してください。
