# Today’s C Challenge: **Bitset Interval Painter**

## 問題
長さ `N`（`1 <= N <= 200000`）のビット列 `A[0..N-1]` をすべて 0 で初期化する。  
`Q` 個の操作（`1 <= Q <= 200000`）を処理せよ。

操作は次の2種類：

- `PAINT l r` : 区間 `[l, r)`（0-index、右端は含まない）のビットを **1 にする**
- `COUNT l r` : 区間 `[l, r)` の **1 の個数**を出力する

## 入力形式
```
N Q
(op1)
(op2)
...
(opQ)
```


各操作は
- `PAINT l r`
- `COUNT l r`
のいずれか。

## 出力形式
`COUNT` のたびに答えを1行で出力。

## 例
入力:
```
8 6
COUNT 0 8
PAINT 2 6
COUNT 0 8
PAINT 4 8
COUNT 3 7
COUNT 0 4
```

出力:
```
0
4
4
2
```


## 制約メモ
- `0 <= l < r <= N`
- `N,Q` ともに大きいので、素直に配列を毎回塗ると間に合わない。

## 要求
- **各操作を概ね `O(log N)` で処理**できる実装を目指すこと。
- C（標準入力）で動くこと。

## ヒント（超短）
- 典型解は **セグ木 + 遅延伝播**（区間を全部1にする代入更新、区間和クエリ）。
- 「この区間はもう全部1」という情報を遅延タグとして持つと速い。

## 追加チャレンジ（余裕があれば）
`PAINT` に加えて `ERASE l r`（0に戻す）も追加してみよう（代入更新の一般形）。
