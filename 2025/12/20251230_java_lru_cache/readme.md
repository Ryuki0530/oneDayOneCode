# 今日の課題（Java・手短）: 「LRUキャッシュ（最小実装）」

## 問題
`LRU (Least Recently Used) キャッシュ`を実装してください。  
容量 `capacity` を超えたら「最後に使われたのが最も古い要素」を自動で捨てます。

次の2つの操作をサポートすること：

- `put(key, value)`：キーに値を保存（同じキーなら上書き。上書き/参照された要素は“最新”扱い）
- `get(key)`：値を取得。存在しない場合は `-1` を返す（取得できた場合、その要素は“最新”扱い）

## 制約
- `capacity` は 1 以上
- キーと値は `int`
- 速度は `get/put` ともに **平均 O(1)** を目標（＝HashMap + 双方向リストの定番構成）

## 入力/出力（この課題では簡易I/O）
標準入力で以下の形式を読み、標準出力に `get` の結果を各行で出してください。

### 入力形式
1行目: `capacity`  
2行目: `Q`（操作回数）  
3行目以降: 操作（`get k` または `put k v`）

### 出力形式
`get` が出るたびに、その返り値を1行出力。

## サンプル
入力:
```
2
8
put 1 10
put 2 20
get 1
put 3 30
get 2
put 4 40
get 1
get 3
```

出力:
```
10
-1
-1
30
```

## 実装要件（これだけ守ればOK）
- `HashMap<Integer, Node>` を使う
- `Node` は `key, value, prev, next` を持つ
- ダミーの `head/tail` を置いて、追加・削除を簡単にする（null地獄を回避）

## ヒント（手短）
- “最新” = リストの先頭側に移動
- 追い出し = リスト末尾（tailの直前）を削除

## ゴール
- `Main.java` 1ファイルで動く形で提出できる状態にする
- サンプルが一致すること
