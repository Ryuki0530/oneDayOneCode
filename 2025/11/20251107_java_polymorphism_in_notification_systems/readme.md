# Java課題：通知システムのポリモーフィズム

## 概要
異なる通知手段（メール、SMS、アプリ内通知など）を共通のインターフェースで扱えるようにしてください。  
ユーザーの通知設定に応じて、実際にどの通知方法が使われるかを動的に切り替えます。

## 要件
1. `Notifier` インターフェースを定義し、共通メソッド `void notifyUser(String message)` を持たせる。  
2. 以下の3つのクラスを定義する：
   - `EmailNotifier`：`notifyUser` で「メールで通知: {message}」と出力
   - `SMSNotifier`：`notifyUser` で「SMSで通知: {message}」と出力
   - `AppNotifier`：`notifyUser` で「アプリ通知: {message}」と出力
3. `Main` クラスで次の動作を実現する：
   - `Notifier` 型の変数に任意の通知クラスを代入し、`notifyUser` を呼び出す。
   - どの通知方法が選ばれても同じ呼び出し構文で動作すること。

## 入出力例
入力なし、実行結果例：
```
メールで通知: セールが始まりました！
SMSで通知: セールが始まりました！
アプリ通知: セールが始まりました！
```
