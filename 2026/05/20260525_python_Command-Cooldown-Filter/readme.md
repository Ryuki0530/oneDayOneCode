# Daily Python Practice: Command Cooldown Filter

## 問題

ロボットやIoT機器では、同じコマンドが短時間に何度も送られると危険な場合があります。

そこで、各コマンドに対して「クールダウン時間」を設けます。

時刻 `t` にコマンド `cmd` が送られてきたとき、同じ `cmd` が最後に採用された時刻から `K` 秒未満であれば、そのコマンドは無視します。

`K` 秒以上経っていれば採用します。

与えられたコマンド列から、採用されたコマンドだけを順番に出力してください。

## 入力

```text
N K
t1 cmd1
t2 cmd2
...
tN cmdN
```

## 条件

- `1 <= N <= 100000`
- `0 <= K <= 100000`
- `0 <= ti <= 1000000000`
- `cmd` は英小文字のみからなる文字列
- 入力される時刻 `ti` は昇順で与えられる

## 出力

採用されたコマンドを、次の形式で出力してください。

```text
M
t cmd
t cmd
...
```

`M` は採用されたコマンド数です。

## 入力例

```text
7 5
0 move
2 move
3 stop
5 move
8 stop
9 move
12 move
```

## 出力例

```text
5
0 move
3 stop
5 move
8 stop
12 move
```

## 説明

- `0 move` は最初なので採用
- `2 move` は前回の `move` から2秒しか経っていないので無視
- `5 move` は前回の `move` から5秒経っているので採用
- `9 move` は前回の `move` から4秒しか経っていないので無視
- `12 move` は前回の `move` から7秒経っているので採用

## 実装のポイント

- コマンドごとの最後の採用時刻を `dict` で管理する
- 入力数が多いので `input()` より `sys.stdin.readline()` を使うとよい
- 採用した結果はリストに保存して、最後にまとめて出力する
