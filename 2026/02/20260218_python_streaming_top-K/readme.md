# 今日の課題: **Streaming Top-K（メモリ節約版）**

大量の数値が1行ずつ入力される状況を想定し、**「いままでに出てきた値の上位K個」**を、すべて保存せずに管理するプログラムを書いてください。

## 目的
- `heapq`（ヒープ）を使って **O(N log K)** で処理する
- 入力全体をリストに溜めず、**ストリーム処理**っぽく書く（メモリ節約）

---

## 問題
標準入力から `N K` が与えられ、その後 `N` 行に整数 `a_i` が与えられる。

あなたのプログラムは、入力を読み終えた時点で **大きい順に上位K個**を出力せよ。

### 制約
- 1 ≤ K ≤ N ≤ 2×10^5
- -10^9 ≤ a_i ≤ 10^9
- 速度とメモリを意識（Pythonで普通に間に合う設計に）

---

## 入力形式
```
N K
a_1
a_2
...
a_N
```


## 出力形式
- 上位K個を **大きい順** に1行ずつ出力

---

## 例
### 入力
```
7 3
5
1
9
3
9
2
8
```

### 出力
```
9
9
8
```



---

## 実装ヒント（使ってOK）
- `heapq` は **最小ヒープ**なので、「サイズKの最小ヒープ」を作っておくと良い  
  - まだK個未満なら push  
  - K個以上なら `heap[0]`（最小）より大きい時だけ入れ替え（`heapreplace` など）
- 最後はヒープの中身を取り出して降順ソートして出力

---

## 追加チャレンジ（余裕があれば）
1. 同じ値が何回出てもそのまま扱う（重複OK）でOK  
2. さらに余裕があれば「上位K個の合計」も最後に出すモードを `--sum` みたいに用意（任意）
