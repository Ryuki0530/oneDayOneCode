# OneDayOneCode: C Optimization Benchmark Lab

## 目的

C言語では、コンパイル時に最適化レベルを指定できる。

今回は、同じCプログラムを次の2種類でコンパイルし、実行時間の差を比較する。

- -O0 : 最適化なし
- -O2 : 一般的な最適化あり

同じ処理・同じ出力結果でも、コンパイル時の最適化によって実行時間が変わることを体験する。

---

## 課題

大量の整数データに対して、何度も計算処理を行うCプログラムを作成せよ。

プログラムは以下を満たすこと。

1. int 型の配列を動的確保する
2. 配列に疑似乱数のような値を入れる
3. 配列全体に対して重い計算を複数回行う
4. 計算結果として checksum を表示する
5. 実行時間を表示する
6. -O0 と -O2 でそれぞれコンパイルし、実行時間を比較する

---

## 作成するファイル

main.c

---

## コンパイル例

最適化なし:

gcc main.c -O0 -o main_O0

最適化あり:

gcc main.c -O2 -o main_O2

---

## 実行例

Linux / WSL / Ubuntu の場合:

./main_O0
./main_O2

Windows の PowerShell の場合:

.\main_O0.exe
.\main_O2.exe

## 注意

- checksum の値は -O0 と -O2 で一致する必要がある
- 実行時間だけが変わることを確認する
- 実行時間が短すぎる場合は、REPEAT の値を増やす
- 実行時間が長すぎる場合は、N または REPEAT の値を減らす
