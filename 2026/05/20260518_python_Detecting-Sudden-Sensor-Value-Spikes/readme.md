# Python課題：センサ値の急上昇検知

## 概要

ロボットやIoT機器では、センサ値が急に大きく変化したときに異常として検知したい場合があります。

今回は、直近 `W` 個の平均値と現在の値を比較し、現在の値が平均より `T` 以上大きい場合に「急上昇」と判定するプログラムを作成してください。

## 入力

```text
N W T
a1 a2 a3 ... aN
```

- `N`：センサ値の個数
- `W`：比較に使う直近データ数
- `T`：急上昇と判定する差
- `ai`：i番目のセンサ値

## 条件

```text
1 <= N <= 200000
1 <= W <= N
0 <= T <= 100000
0 <= ai <= 100000
```

## 判定ルール

i番目の値 `ai` について、直前の `W` 個の平均値と比較する。

ただし、直前に `W` 個のデータが存在しない場合は判定しない。

つまり、判定対象は `i > W` のデータのみ。

以下を満たす場合、i番目のデータを急上昇として出力する。

```text
ai >= 直前W個の平均値 + T
```

浮動小数点の誤差を避けるため、次の形で比較してもよい。

```text
ai * W >= 直前W個の合計 + T * W
```

## 出力

急上昇と判定されたデータの番号を、1行に1つずつ出力してください。

データ番号は1始まりです。

急上昇が1つもない場合は、次を出力してください。

```text
OK
```

## 入力例1

```text
8 3 10
10 12 11 25 13 40 41 20
```

## 出力例1

```text
4
6
```

## 説明

4番目の値は `25`。

直前3個は `10, 12, 11` で、合計は `33`。

```text
25 * 3 >= 33 + 10 * 3
75 >= 63
```

なので急上昇。

6番目の値は `40`。

6番目を判定する直前3個は、5番目までの直近3個である `11, 25, 13` です。

```text
40 * 3 >= 49 + 10 * 3
120 >= 79
```

なので急上昇。

## 実装方針

- 直近W個の合計を変数で管理する
- 新しい値を判定した後、ウィンドウを1つ進める
- `sum()` を毎回使うと遅いので使わない
- 計算量は `O(N)` を目指す

## 使用してよいもの

- `sys.stdin`
- `collections.deque`

## 目標

- 標準入力から読み込む
- 大きな入力でも高速に処理できる
- 浮動小数点を使わずに平均比較を行う

## ヒント

平均を小数で出さず、次のように整数だけで比較すると安全です。

```python
a[i] * W >= window_sum + T * W
```
