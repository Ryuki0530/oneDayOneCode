# 今日の課題（Python・手短）

## 問題：ログの「ピーク同時数」を数える（スライディングウィンドウ）

サーバのアクセスログが **「秒」単位で** 与えられます。  
各アクセスはその秒に開始し、**継続時間 D 秒** だけ「アクティブ」とみなします（開始秒を含み、終了境界は含まない）。

つまり、アクセスが時刻 `t` に来たら、区間 `[t, t + D)` のあいだアクティブです。

あなたの仕事は、全期間を通して **同時にアクティブなアクセス数の最大値** を求めることです。

---

## 入力
- 1行目：整数 `N D`  
  - `1 <= N <= 200000`
  - `1 <= D <= 10^9`
- 2行目：整数 `t1 t2 ... tN`  
  - 各 `ti` はアクセス開始時刻（秒）
  - `0 <= ti <= 10^9`
  - **昇順とは限らない**（バラバラでも来る）

---

## 出力
- 1行：最大同時アクティブ数

---

## 例1
入力:
```
6 3
1 2 2 3 5 6
```


出力:
```
4
```

説明（D=3）：  
- t=1 は [1,4)
- t=2 は [2,5) が2件
- t=3 は [3,6)
時刻3の瞬間には 1件(t=1) + 2件(t=2) + 1件(t=3) = 4件アクティブ。

---

## 例2
入力:
```
5 2
10 10 12 13 14
```

出力:
```
2
```

---

## 制約上の狙い
- O(N log N) まで（ソートはOK）
- できれば **二本ポインタ（スライディングウィンドウ）** で O(N)

---

## ヒント（読み飛ばしてOK）
開始時刻をソートして、左端 `l` を「もう非アクティブになったもの」まで進めながら、
各 `r` について `ti[r] - ti[l] < D` を満たす範囲の長さを最大化するとよい。

---
