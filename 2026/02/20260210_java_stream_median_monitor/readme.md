# 今日の課題: **Stream Median Monitor**

## 問題
数列が先頭から1つずつ追加されていくストリームがある。  
各要素が追加されるたびに、その時点までの **中央値** を出力せよ。

- 要素数が奇数のとき：真ん中の値
- 要素数が偶数のとき：真ん中2つの **小さい方**（lower median）

## 入力
- 1行目: 整数 `N`（1 ≤ N ≤ 200000）
- 2行目: 整数 `A1 A2 ... AN`（各 `Ai` は -10^9 ～ 10^9）

## 出力
`N` 行出力。  
i行目に、`A1..Ai` の中央値（lower median）を出力。

## 例
入力:
5
1 5 2 10 6

出力:
1
1
2
2
5

## 制約・狙い
- `O(N log N)` を目標（毎回ソートすると間に合わない）
- ヒント: **2つのヒープ**（max-heap と min-heap）で「左半分」「右半分」を管理する

## 実装ヒント（超要点）
- 左: max-heap（中央値候補を含む小さい側）
- 右: min-heap（大きい側）
- 追加のたびに:
  1. どちらかに入れる
  2. サイズ条件を調整（左のサイズが右以上、かつ差は高々1）
  3. 答えは `左.peek()`（lower median）
