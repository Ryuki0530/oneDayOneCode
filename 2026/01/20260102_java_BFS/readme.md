# 今日のJava課題：最短距離（BFS）で迷路を解く（手短版）

## 問題
`.`（通路）と `#`（壁）で表された迷路が与えられます。  
スタート `S` からゴール `G` まで **上下左右** に移動できるとして、**最短移動回数**を出力してください。  
到達不能なら `-1` を出力してください。

## 入力形式
- 1行目：`H W`（高さ・幅）
- 続くH行：長さWの文字列（迷路）

文字の意味：
- `S`：開始地点（1つ）
- `G`：目的地点（1つ）
- `.`：通路
- `#`：壁

## 出力
- `S` から `G` の最短手数（移動回数）
- 到達できない場合は `-1`

## 制約（目安）
- `1 <= H, W <= 200`

## 例
### 入力
5 7
#######
#S...##
#.....#
#...#G#
#######


### 出力
8

## 実装条件（最低限）
- BFS（幅優先探索）を使うこと
- `dist[h][w]` を `-1` 初期化して距離管理
- キュー（`ArrayDeque<int[]>` など）で探索

## ヒント（超短）
- `S` の位置を探してキューに入れる
- 4方向を見て、壁でなく未訪問なら距離+1で更新
- `G` の距離を出力（未訪問なら -1）
