# 今日のC課題：ログの「最頻エラーコード」をリアルタイム集計せよ（タイブレーク付き）

## 問題
サーバの簡易ログが標準入力から与えられます。各行は次の形式です。

- `t code`
  - `t` は整数のタイムスタンプ（秒）
  - `code` は整数のエラーコード（例：404, 500 など）

あなたのプログラムは、**直近W秒のスライディングウィンドウ**（時刻が `T-W+1` 〜 `T` に含まれるログ）における  
**最頻（出現回数最大）のエラーコード**を、各入力行を処理するたびに出力してください。

### タイブレーク規則
最頻が複数ある場合は、**数値が小さい code** を選びます。

## 入力
- 1行目：整数 `W`（1 ≤ W ≤ 10^6）
- 2行目以降：複数行の `t code`（EOFまで）
  - `t` は非減少（同じ時刻が連続してもよい）
  - 行数は最大 2×10^5 程度を想定

## 出力
各ログ行を処理するたびに、その時点の直近W秒ウィンドウでの「最頻エラーコード」を1行で出力。

## 例
入力：
```
3
1 500
2 404
3 500
4 404
4 404
```

出力：
```
500
404
500
404
404
```

（説明：W=3なので、t=4時点では [2..4] のログだけが対象）

## 要求
- できるだけ高速に（O(N log N) くらいを目標）
- 標準C（C11想定）。外部ライブラリ禁止。

## ヒント（使っていい）
- ウィンドウから外れる古いログを捨てるために「キュー（配列+先頭index）」が便利。
- `code -> 出現回数` の管理が必要（ハッシュが欲しくなるが、Cなので自作すると楽しい）。
- 「最頻をすぐ答える」には、頻度ごとにまとめる/優先度付き構造を考える（例：最大ヒープ、または“怠惰削除”付きヒープ）。

## 提出物
- `main.c` のみ（標準入力→標準出力）
