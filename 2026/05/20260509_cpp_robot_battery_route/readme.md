# 今日の課題：Robot Battery Route

## 概要

バッテリー付きロボットをグリッド上で移動させるシミュレーションを作成してください。

## 問題文

縦 `H`、横 `W` のマス目があります。

各マスは次のいずれかです。

- `.` : 通路
- `#` : 壁
- `S` : ロボットの開始位置
- `C` : 充電マス

ロボットには最大バッテリー量 `B` があり、最初はバッテリーが満タンです。

長さ `N` の移動命令列が与えられます。

命令は以下の4種類です。

- `U` : 上へ移動
- `D` : 下へ移動
- `L` : 左へ移動
- `R` : 右へ移動

各命令を1つ実行するたびに、移動できたかどうかに関係なくバッテリーを `1` 消費します。

ただし、命令を実行する前にバッテリーが `0` の場合、その時点で処理を終了します。

移動先が範囲外、または壁 `#` の場合、ロボットはその場にとどまります。

移動後、ロボットが充電マス `C` にいる場合、バッテリーは最大値 `B` まで回復します。

最終的なロボットの位置、残りバッテリー、成功した移動回数を出力してください。

## 入力形式

    H W B N
    grid_1
    grid_2
    ...
    grid_H
    commands

## 出力形式

    row col battery success_count

- `row`, `col` は最終位置を `1-indexed` で出力すること
- `battery` は残りバッテリー
- `success_count` は実際にマスを移動できた回数

## 制約

- `1 <= H, W <= 100`
- `1 <= B <= 1000`
- `1 <= N <= 100000`
- `commands` は `U`, `D`, `L`, `R` のみからなる
- `S` は必ず1つだけ存在する

## 入力例

4 5 5 8
S....
.##C.
.....
..#..
RRDDRRUU

## 出力例

1 4 0 3 

## 考え方のヒント

- グリッドは `vector<string>` で管理すると扱いやすいです。
- 現在位置を `r`, `c` で持ちます。
- 命令ごとに移動先 `nr`, `nc` を計算します。
- 移動できる条件は「範囲内」かつ「壁ではない」ことです。
- バッテリーが `0` になったら、残りの命令は処理しません。
