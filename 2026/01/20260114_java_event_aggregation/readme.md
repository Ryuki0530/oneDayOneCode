# 今日のJava課題：イベント集計（スイープライン）

## 問題
あなたは大学のイベント管理システムを作っています。  
各イベントは「開始時刻 `s`」と「終了時刻 `t`」を持ち、区間 `[s, t)`（開始は含む・終了は含まない）として扱います。

次の3つを計算してください。

1. **最大同時開催数**（同じ時刻に開催中のイベント数の最大値）
2. **最大同時開催数になっていた合計時間**
3. **あるクエリ時刻 `q` における開催中イベント数**（複数クエリ）

---

## 入力形式
- 1行目：`N M`  
  - `N` = イベント数  
  - `M` = クエリ数
- 次のN行：`s t`（整数、`s < t`）
- 次のM行：`q`（整数）

---

## 出力形式
3行で出力してください。

1行目：最大同時開催数 `K`  
2行目：最大同時開催数だった合計時間 `L`  
3行目：各クエリの答えを改行区切りで（M行）

---

## 制約
- `1 <= N, M <= 200000`
- `0 <= s, t, q <= 10^9`
- 時刻は整数
- 区間は **[s, t)**（ここ超重要：tちょうどの瞬間は「終了済み」扱い）

---

## 例
### 入力
```
3 4
1 5
2 6
4 7
0
1
5
6
```

### 出力
```
3
1
0
1
1
1
```


（時刻4〜5の1時間だけ同時開催が3）

---

## 要求
- `O((N+M) log (N+M))` を狙う（素直に全部の時刻を舐めるのは禁止）
- Javaで実装（標準入力・標準出力）
- 大量入力なので `BufferedInputStream` + 自前パーサ推奨（`Scanner`は重い）

---

## ヒント
- イベントは「開始 +1」「終了 -1」の**差分**にできる
- 最大同時開催時間 `L` は、座標圧縮して隣り合う時刻差 `dx` に「その区間の開催数」を掛けるイメージ
- クエリ `q` は「qまでの累積和」を取ればよい（`[s,t)`なので終了側の扱いに注意）
