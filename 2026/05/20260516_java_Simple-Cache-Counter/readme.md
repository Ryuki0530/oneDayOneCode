# 今日の課題：Simple Cache Counter

## 概要

小さなキャッシュの動作をシミュレーションしてください。

キャッシュには最大 `K` 個まで整数IDを保存できます。
`N` 回のアクセスがあり、それぞれ整数ID `A_i` が与えられます。

各アクセスについて、次の処理を行います。

- すでにキャッシュ内にあるIDなら `HIT`
- キャッシュ内にないIDなら `MISS`
  - キャッシュに空きがあれば追加する
  - 空きがなければ、最も古く追加されたIDを削除してから追加する

すべてのアクセス後、`HIT` の回数と `MISS` の回数を出力してください。

## 入力

```text
N K
A_1 A_2 ... A_N
```

## 出力

```text
hit_count miss_count
```

## 制約

- 1 <= N <= 100
- 1 <= K <= 10
- 1 <= A_i <= 1000

## 入力例

```text
8 3
1 2 3 1 4 2 5 1
```

## 出力例

```text
2 6
```

## 説明

キャッシュサイズは3です。

1, 2, 3 は MISS で追加されます。
次の 1 はすでにあるので HIT です。
4 は MISS で、最も古い 1 を削除して追加します。
以降も同様に処理します。

## 実装ヒント

Javaでは次のような構成にすると実装しやすいです。

```java
Queue<Integer> queue = new LinkedList<>();
Set<Integer> cache = new HashSet<>();
```

- `Set` でキャッシュ内に存在するか確認する
- `Queue` で古い順にIDを管理する
- 削除するときは `queue.poll()` で一番古いIDを取り出す
