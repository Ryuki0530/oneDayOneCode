# 【Java 課題】ログに含まれる時刻の整合性チェックツール

## 問題文

あるシステムのログファイル `system.log` には、下記のように  
`[HH:MM:SS] メッセージ本文`  
という形式の行が多数含まれている。

例：
```
[12:03:10] System boot
[12:03:15] Connection established
[12:03:12] Warning: something happened
```

ただし、ログは本来 **時刻が昇順**になるように並ぶべきである。  
しかし混乱があり、一部の行が **時刻が逆転してしまっている**可能性がある。

### 要求仕様

1. `system.log` を読み込み、各行の先頭の時刻 `[HH:MM:SS]` を解析する。
2. 行ごとの時刻が **昇順であるか**をチェックし、  
   **逆転している行（前の行より時間が早くなっている行）を全て報告**する。
3. 出力形式は以下とする：
```
Line X is out of order: HH:MM:SS
```

4. ログが完全に昇順なら：
```
All timestamps are in correct order.
```
### 入出力

- 入力：同じディレクトリにある `system.log`
- 出力：コンソールへのテキスト出力

---

## 追加条件

- Java 標準ライブラリのみ使用。
- ファイルが存在しない場合はエラー表示して終了。

---

## 例の動作
```log
[12:00:00] start
[12:00:05] ok
[11:59:59] error!
``` 

実行結果：
```
Line 3 is out of order: 11:59:59
```
