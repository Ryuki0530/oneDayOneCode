# 今日の課題：Multi-Source Log Merger（複数ログの時系列マージ）

## 問題
複数のサービスがそれぞれログを出しています。  
あなたは **K個のログ列** をまとめて、時刻順に1本のログとして出力するツールを書きます。

各ログ列はすでに **時刻の昇順にソート済み** です。  
ただし、異なるログ列同士では時刻が入り混じります。

あなたの仕事は、全ログを **時刻の昇順** にマージして出力することです。

---

## 入力形式
- 1行目：`K`（ログ列の数）
- 続くKブロック：
  - 1行目：`N_i`（i番目ログ列の行数）
  - 続く`N_i`行：`t message`
    - `t` は整数（時刻）
    - `message` は空白を含まない文字列

---

## 出力形式
全ログを次のルールで並べて、1行ずつ出力せよ。

### 並び順ルール
1. `t`（時刻）が小さいものが先
2. `t` が同じなら **sourceId（どのログ列か：0..K-1）** が小さいものが先
3. `t` と sourceId が同じなら **そのログ列での出現順** が先（安定）

出力は次の形式：
`sourceId t message`

---

## 制約
- `1 <= K <= 20`
- `0 <= N_i <= 200000`（合計Nは最大 200000 程度を想定）
- `t` は `0 <= t <= 10^9`
- `message` は英数字と記号（空白なし）

---

## 例
### 入力
```
3
3
1 A
4 B
9 C
2
2 X
4 Y
3
1 P
4 Q
10 R
```

### 出力
```
0 1 A
2 1 P
1 2 X
0 4 B
1 4 Y
2 4 Q
0 9 C
2 10 R
```


---

## 実装要件（重要）
- **優先度付きキュー（PriorityQueue）** を使って解くこと。
- メモリ爆発を避けるため、全ログを1つの巨大配列にまとめてソートするのは禁止（やるなら理由が必要）。
- 各ログ列から「次に出す候補」だけをキューに入れてマージする、いわゆる **k-way merge** を行うこと。

---

## ヒント（使うモジュール）
### PriorityQueue の使い方（超要点）
- `PriorityQueue<Node> pq = new PriorityQueue<>((a,b) -> {...});`
- 比較は `(t, sourceId, indexInSource)` のタプル順に。

### 典型手順
1. 各ログ列の先頭要素をpqに入れる（空ならスキップ）
2. pqから最小を取り出して出力
3. 取り出した要素が属するログ列の「次の要素」をpqへ入れる
4. pqが空になるまで繰り返す

---

## 追加チャレンジ（時間があれば）
- `message` に空白を含む版（行末までをmessageとして扱う）に拡張せよ。
- 出力を `t` ごとにまとめてバッチ表示（同時刻のログを1グループで出す）してみよ。
