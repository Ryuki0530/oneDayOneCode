# 今日の課題（C++）: ログの「ウィンドウ集計」ツール

## 問題
標準入力からアクセスログ（タイムスタンプ付き）を読み込み、**スライディングウィンドウ（固定幅の時間窓）**ごとにリクエスト数を集計して出力してください。

ログは 1 行につき 1 リクエストで、形式は次の通りです。
```
YYYY-MM-DD HH:MM:SS <path>
```

例:
```
2025-12-19 12:00:01 /index
2025-12-19 12:00:05 /api
2025-12-19 12:00:12 /index
``` 

集計したいのは「最後の W 秒間に何件あったか」を、各ログ行の時刻を基準に計算した値です。

---

## 入力
1 行目:
W

- `W` はウィンドウ幅（秒）を表す整数（1〜10^6）

2 行目以降:
- ログ行が 0 行以上続く（EOF まで）

---

## 出力
各ログ行について 1 行出力し、次を表示してください。

```
YYYY-MM-DD HH:MM:SS <path> <count>
```

ここで `<count>` は、その行の時刻 `t` に対して **(t - W + 1) 〜 t** の範囲（両端含む）にあるログ行の件数です。
（つまり「同じ秒」も含み、W=1 なら同秒の件数になります）

---

## 制約・注意
- 入力ログは **時刻の昇順（同時刻は入力順）** で与えられるとします。
- `<path>` は空白を含まない文字列。
- 実装は `O(N)` を目指してください（各行で全探索しない）。
- C++17 で実装してください。

## 例

### 入力
```
10
2025-12-19 12:00:01 /a
2025-12-19 12:00:05 /b
2025-12-19 12:00:12 /c
2025-12-19 12:00:12 /d
2025-12-19 12:00:20 /e
```

### 出力
```shell
2025-12-19 12:00:01 /a 1
2025-12-19 12:00:05 /b 2
2025-12-19 12:00:12 /c 3
2025-12-19 12:00:12 /d 4
2025-12-19 12:00:20 /e 3

（12:00:20 の直近10秒は 12:00:11〜12:00:20 なので、12:00:12×2 と 12:00:20 の計 3 件）
```

## 実装ヒント（方針だけ）
- 直近 W 秒に入るログの時刻だけを `queue` などで保持する。
- 新しいログ時刻 `t` が来たら、範囲外（< t-W+1）のものを先頭から捨てる。
- 残った要素数が `<count>`。


## 追加（テストしやすいポイント）
- ログが 0 行の場合は出力なし。
- 同じ秒が連続するケースでカウントが増えていくことを確認。
- W が大きいケース（全部含まれる）も確認
