# 今日のPython課題（手短）：LRUキャッシュ

## 問題
容量 `C` の **LRU（Least Recently Used）キャッシュ**を実装してください。

次の `Q` 個の操作を処理し、`get` の結果を出力します。

- `put k v` : キー `k` に値 `v` を保存する  
  - すでに存在するなら値を更新し、そのキーを「最新」にする
  - 容量を超える場合は「最も最近使われていない」キーを1つ削除する
- `get k` : キー `k` の値を取得する  
  - 存在するならその値を出力し、そのキーを「最新」にする
  - 存在しないなら `-1` を出力する

## 入力形式
1行目: `C Q`  
続く `Q` 行: `put k v` または `get k`

## 出力形式
各 `get` 操作ごとに1行、結果を出力。

## 制約（目安）
- `1 <= C <= 2*10^5`
- `1 <= Q <= 2*10^5`
- `k, v` は int

## 例
入力:
C=2, Q=8
put 1 10
put 2 20
get 1
put 3 30
get 2
put 4 40
get 1
get 3

出力:
10
-1
-1
30

## 要件
- **O(1) 平均**で `get/put` を処理できること（dict + 双方向リスト相当）
- Python標準ライブラリのみでOK（`collections.OrderedDict` を使っても良い）
