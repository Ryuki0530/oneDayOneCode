# One Day One Code - Java編

## 問題名
**Recent Message Counter**

## 問題概要
チャットアプリのログが時系列順に与えられます。  
各メッセージには「送信時刻」と「ユーザー名」が含まれています。

あなたの仕事は、各メッセージが送信された時点で、その直前 **T 秒以内** にそのユーザーが何回発言していたかを求めることです。  
※現在のメッセージ自身も回数に含めます。

---

## 入力

```
N T
time1 user1
time2 user2
...
timeN userN
```

- **N** : ログ件数  
- **T** : 対象とする秒数  
- **timei** : i件目の発言時刻（整数）  
- **useri** : i件目のユーザー名（英小文字）

---

## 出力

各メッセージについて、その時点で直前 **T 秒以内** にそのユーザーが発言した回数を1行ずつ出力してください。

---

## 例

### 入力例

```
7 5
1 alice
2 bob
4 alice
7 alice
8 bob
10 alice
12 bob
```

### 出力例

```
1
1
2
2
2
2
2
```

---

## 例の説明

- `1 alice`  
    区間は [-4, 1]  
    alice の発言は 1 のみ → **1**

- `4 alice`  
    区間は [-1, 4]  
    alice の発言は 1, 4 → **2**

- `7 alice`  
    区間は [2, 7]  
    alice の発言は 4, 7 → **2**

- `10 alice`  
    区間は [5, 10]  
    alice の発言は 7, 10 → **2**

---

## 制約

- 1 <= N <= 200000
- 1 <= T <= 10^9
- 0 <= timei <= 10^9
- timei <= time(i+1)  （時刻は昇順、同時刻もあり）
- useri は英小文字からなる長さ1以上20以下の文字列

---

## ヒント

- 各ユーザーごとに最近の発言時刻だけを保持すると効率よく解けます。
- `HashMap<String, Deque<Integer>>` を使うと実装しやすいです。
- 今の時刻 `time` に対して、`time - T` より小さい時刻はもう不要です。

---

## 目標

- HashMap と Deque に慣れる
- 時系列データの「古い要素を捨てる」処理を練習する
- 実務っぽい「直近 N 秒集計」の考え方に触れる

---

## 発展課題

1. 各時点での全ユーザー合計発言数も出力する
2. 直前 T 秒以内で最も発言回数の多いユーザーを求める
3. 同時刻の発言が大量にあるケースでも正しく動くことを確認する
