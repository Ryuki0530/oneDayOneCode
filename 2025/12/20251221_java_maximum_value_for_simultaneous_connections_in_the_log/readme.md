# 今日の課題（Java）: ログの「同時接続数」最大値を求める

サーバのアクセスログ（簡略版）が標準入力から与えられます。  
各行は「開始時刻」と「終了時刻」を表し、その区間は **[start, end)**（開始は含む／終了は含まない）とします。

同時に処理中（区間内）となるリクエスト数の **最大値** を出力してください。

---

## 入力形式

1行目: 整数 `N`（ログ行数）  
続く `N` 行: `start end`（整数、`0 <= start < end <= 10^9`）

---

## 出力形式

同時接続数の最大値を表す整数を1行で出力。

---

## 制約

- `1 <= N <= 200000`
- 時刻は秒などの整数として扱う
- 区間は **[start, end)**（end は重ならない側）

---

## 例

### 入力
5
1 4
2 5
4 6
3 7
6 8

### 出力
3

### 説明（ざっくり）
時刻3のあたりで `[1,4)`, `[2,5)`, `[3,7)` が同時に走って最大3になります。  
`[1,4)` と `[4,6)` は **4で重ならない**（endが含まれない）点に注意。

---

## 実装ヒント（短い）

「開始で +1」「終了で -1」のイベントにして、時刻順に足し引きして最大を取る（いわゆる sweep / line sweep）。  
同じ時刻に開始と終了がある場合は、**終了（-1）を先に処理**すると `[start, end)` の仕様に一致します。

---

## 期待するもの

- `O(N log N)` で通る実装
- 入力が大きいので `BufferedInputStream` / `FastScanner` 推奨
