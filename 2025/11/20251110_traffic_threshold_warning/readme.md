# 1Day1Code–Java: 通信量しきい値警告（スライディングウィンドウ）

## 問題概要
あるサービスのアクセスログが時刻と転送バイト数の列として記録されています。  
直近 `W` 分間の通信量合計がしきい値 `T` を**初めて**超えた（またはちょうど等しくなった）時刻を「警告時刻」として検出してください。  
ここで「直近 `W` 分間」とは、ある時刻 `t` に対し `[t - W + 1分, t]` の区間に含まれるログのバイト数合計を意味します（時刻は「分」単位の閉区間で数えます）。

複数のログが同一分に来ることもあります。入力されるログ時刻は**非減少**（同一時刻の並びは任意）です。

## 入力
標準入力から次の形式で与えられます。
```
N W T
t1 b1
t2 b2
:
tN bN
```

- `N`（1 ≤ N ≤ 2×10^5）：ログ行数  
- `W`（1 ≤ W ≤ 10^5）：監視ウィンドウ（分）  
- `T`（1 ≤ T ≤ 10^15）：しきい値（バイト）
- 各 `ti` は `YYYY-MM-DD HH:MM` 形式の文字列（先頭ゼロあり、UTCやタイムゾーンは考えない）
- 各 `bi` はその分の転送バイト数（0 ≤ `bi` ≤ 10^9）
- `t1 ≤ t2 ≤ … ≤ tN` が保証されます。

## 出力
- 直近 `W` 分間の合計が `T` 以上になった**最初の**時刻（入力に現れる時刻のいずれか）を `YYYY-MM-DD HH:MM` 形式で1行に出力してください。  
- そのような時刻が一度も存在しない場合は `OK` と出力してください。

> **注意**：  
> 「最初の時刻」とは、時系列に沿ってはじめて `合計 ≥ T` を満たす行の `ti` を指します。  
> すでに `合計 ≥ T` の状態が続いている間の後続行は出力対象ではありません（1回だけ）。

## 例1
**入力**
```
5 3 1000
2025-11-10 14:00 200
2025-11-10 14:01 600
2025-11-10 14:03 300
2025-11-10 14:03 200
2025-11-10 14:05 100
```

**解説（要点）**  
- `W=3` なので、各行の時刻 `t` に対し `[t-2分, t]` を合計。  
- 14:03 の2行目処理時点で、区間 `[14:01,14:03]` の合計は `600 + 300 + 200 = 1100` となり、初めて `T=1000` を超える。

**出力**
```
2025-11-10 14:03
```

## 例2
**入力**
```
4 5 1000000
2025-01-01 00:00 10000
2025-01-01 00:01 20000
2025-01-01 00:03 30000
2025-01-01 00:06 40000
```

**出力**
```
OK
```
## 制約と実装のヒント
- `N` は最大 2×10^5 なので、各行で直近 `W` 分の合計を**愚直に再計算**すると TLE になり得ます。  
  2ポインタ（デック/キュー）またはインデックスで先頭を追い出す**スライディングウィンドウ**で **O(N)** を目指しましょう。
- 時刻は分解能が「分」なので、`YYYY-MM-DD HH:MM` を「通算分」（例：`epochMinutes`）に変換すると比較・減算が容易です。  
  例：`LocalDateTime` → `toEpochSecond(ZoneOffset.UTC) / 60` など。標準ライブラリのみで実装可（`java.time` パッケージ）。
- 同一分に複数行が来る場合は、それぞれ別行として足し込みます。  
- ウィンドウ境界は**閉区間** `[t-W+1, t]`。たとえば `W=1` のとき、その分単体だけが対象です。

## 仕様確認チェックリスト
- [ ] 入力のパース：空白区切りで `ti`（2トークン）と `bi`（整数）を読む  
- [ ] `ti` の**非減少**前提で先頭を追い出す処理  
- [ ] 合計が `T` に達した**最初の行**で即出力して終了  
- [ ] 最後まで到達して未達なら `OK` を出力  
- [ ] 64-bit（`long`）で合計・しきい値を扱う

## 発展課題（任意）
1. 同一分の行を事前に集約してから処理すると、理論上の操作回数がさらに減ります。  
2. `W` が非常に大きい場合のメモリ使用量最適化（配列リングバッファ vs デック）を比較してみましょう。  
3. 複数しきい値 `T1, T2, …` に対して、それぞれの到達時刻を1走査で同時に検出する拡張。
