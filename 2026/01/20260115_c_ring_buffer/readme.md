# 今日のC課題：リングバッファで「直近K個の平均」を高速に出す

## 問題
整数列 `A1..AN` が順に与えられる。各 `Ai` を読み込んだ直後に、

- 直近 `K` 個（まだ `K` 個未満なら、これまで読んだ全て）の平均値

を **小数で** 出力せよ。

ただし、毎回 `K` 個を足し直すのは禁止（TLE想定）。  
**リングバッファ + 走る合計 (running sum)** で `O(N)` を目指すこと。

## 入力
- 1行目：`N K`
- 2行目以降：`A1..AN`（スペース区切り）

## 出力
`i=1..N` について、各行に平均を出力せよ（`double` でOK）。
- 出力誤差は `1e-6` 以内なら正解扱い。

## 制約（目安）
- `1 <= N <= 200000`
- `1 <= K <= 200000`
- `|Ai| <= 10^9`

## 例
入力
```
7 3
1 2 3 4 5 6 7
```

出力
```
1.000000
1.500000
2.000000
3.000000
4.000000
5.000000
6.000000
```


## 実装ヒント
- `long long sum` で合計を管理（オーバーフロー注意）
- `int buf[K]` のリングバッファを用意し、位置 `pos = i % K` に上書き
- `i <= K` の間は足すだけ
- `i > K` になったら、`sum += Ai - buf[pos]` の形で更新
- 平均は `sum / (double)min(i, K)` で出す
