# 今日のPython課題：Pygameで「エアホッケー風・1人用リフレクション」

## 目的
- Pygameの基本（ウィンドウ、入力、更新、描画、FPS制御）を思い出す
- 当たり判定（矩形×円っぽい処理でもOK）と反射の実装に慣れる

---

## 仕様（必須）

### 画面
- ウィンドウサイズ：800×600
- 背景は単色でOK
- FPS：60固定（Clockで制御）

### オブジェクト
- **パドル（自機）**：横長の矩形（例：幅120 高さ15）
  - 画面下側に置く
  - 操作：左右移動（←→ または A/D）
  - 画面外に出ない

- **ボール**：円（例：半径10）
  - 速度ベクトル (vx, vy) を持つ
  - 最初は画面中央付近からスタートして、適当な方向へ動く

### 壁反射
- 左右の壁に当たったら vx を反転
- 上の壁に当たったら vy を反転

### パドル反射（ここが本題）
- ボールがパドルに当たったら **上方向へ跳ね返る**
- さらに、当たった位置で跳ね返り角度が変わるようにする：
  - パドル中心からの相対位置 `t` を `[-1, 1]` に正規化
  - `vx` を `t` に応じて変化させる（例：vx = MAX_VX * t）
  - 速さが極端にならないように上限を持たせる

### ミス判定
- ボールが画面下に出たら「ミス」
  - スコアをリセット or ライフを減らす（どっちでも良い）
  - ボール位置と速度を初期化して再スタート

### 表示
- スコア（生存時間でも跳ね返した回数でもOK）を左上に表示

---

## 追加チャレンジ（余裕があれば）
- [1] パドルに当たるたびにボール速度を少し上げる（上限あり）
- [2] スペースキーで「スタート/一時停止」切り替え
- [3] “壁に当たったらピッ” みたいな効果音（無くてもOK）
- [4] パドルの端に当たると、より鋭く跳ねる（反射角が大きい）

---

## 実装ヒント（考え方だけ）
- **ゲームループの基本形**
  1. イベント処理（終了、キー入力）
  2. 状態更新（座標 += 速度、当たり判定、スコア更新）
  3. 描画（背景→物体→文字）
  4. `clock.tick(60)` でFPS固定

- **当たり判定を簡単にする方法**
  - ボールを「rectにした当たり箱」で近似してもOK（まず動かすのが正義）
  - 余裕があれば「円の中心点」と「パドル矩形」の最近点で判定する
