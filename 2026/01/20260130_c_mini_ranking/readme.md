# 今日のC課題：ストリームの「上位K件」だけを保つ（ミニ・ランキング）

## 問題
標準入力で、整数 `N` と `K`、続いて `N` 個の整数 `a[i]` が与えられる。  
**上位 `K` 個（大きい順）**だけを保持しつつ全入力を処理し、最後に **上位K個を大きい順**で出力せよ。

ただし、同じ値が複数回出た場合も **別個**として数える（多重集合）。

## 入力形式
- 1行目: `N K`
- 2行目以降: `a1 a2 ... aN`（空白区切り、改行は任意）

## 出力形式
- 上位K個を **大きい順**で1行に空白区切りで出力  
- `K > N` の場合は、存在する分（N個）だけ出力

## 制約（目安）
- `1 <= N <= 200000`
- `1 <= K <= 200000`
- `a[i]` は 32bit signed int に収まる
- できれば `O(N log K)` で解きたい（全ソート `O(N log N)` は避ける）

## 例
入力:
```
8 3
5 1 7 7 2 9 3 0
```

出力:
```
9 7 7
```

## ヒント（超短）
- 「上位Kだけ保持」なら **最小ヒープ（min-heap）** を使う。
- サイズがKを超えたら、最小（ヒープ頂点）を捨てる。
- 最後にヒープ中身を取り出して並べ替え（または大きい順に出力）する。

## 実装条件
- C標準ライブラリのみ（`malloc/free` OK）
- ヒープは自前実装（配列でOK）
