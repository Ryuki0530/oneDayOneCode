# 今日のJava課題：最短経路（0-1 BFS）

## 問題
H×W のグリッドが与えられる。各マスは `.`（通れる）か `#`（壁）で、さらに開始 `S` とゴール `G` が1つずつある。

あなたは上下左右に移動できる。ただし移動コストは次の通り：
- `.` / `S` / `G` に入る移動はコスト **0**
- `#` に入る移動はコスト **1**（= 壁を「壊して」進むイメージ）

`S` から `G` までの **最小コスト**（壊す壁の最小数）を出力せよ。

## 入力
- 1行目：`H W`
- 次のH行：長さWの文字列（`.`, `#`, `S`, `G`）

## 出力
- 最小コストを1行で出力

## 制約（目安）
- 1 ≤ H, W ≤ 1000（ただし実装はO(HW)を目標）
- `S`, `G` は必ず存在する

## 例
入力
3 4
S#.#
.##.
...G

出力
1

## ヒント
- 辺の重みが 0/1 だけなので、Dijkstraより **0-1 BFS** が速い。
- `Deque` を使い、重み0なら前にpush、重み1なら後ろにpush。
- dist配列は大きい値で初期化。

## 使ってほしいJava要素
- `ArrayDeque<int[]>`
- 2次元 `int[][] dist`

## クリア条件
- 0-1 BFSで最小コストを求め、正しく出力できること。
