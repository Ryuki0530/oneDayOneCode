# 今日の課題：設定ファイルの重複キー検出ツール

## 概要

アプリケーションの設定ファイルを読み込み、同じキーが複数回定義されていないかを検出するプログラムを作成してください。

設定ファイルは、次のような形式で与えられます。

key=value

同じ key が複数回出現した場合、その key を重複として出力します。

## 入力

1行目に設定行数 N が与えられます。

続く N 行に、設定項目が `key=value` の形式で与えられます。

## 制約

- 1 <= N <= 1000
- key は英小文字、数字、アンダースコア `_` のみ
- value は空白を含まない文字列
- `=` は必ず1つ含まれる

## 出力

重複している key を、最初に重複が発生した順に1行ずつ出力してください。

重複が存在しない場合は、次の文字列を出力してください。

OK

## 入力例1

5
host=localhost
port=8080
mode=debug
port=3000
host=127.0.0.1

## 出力例1

port
host

## 入力例2

3
host=localhost
port=8080
mode=debug

## 出力例2

OK

## 実装方針

- `unordered_set<string>` を使って、すでに登場した key を管理する
- 重複 key を二重に出力しないように、別の `unordered_set<string>` で出力済み重複 key を管理する
- `find('=')` を使って key 部分だけを切り出す

## 目標

実務では、設定ファイルのミス検出やデプロイ前チェックのような小さな検証ツールを作る場面があります。
今回は、その基礎として「設定値の読み取り」「文字列分割」「重複検出」を練習します。
